# 성능 측정 스크립트 실행 가이드

본 폴더에는 성능 측정 문서(단건 및 다중 부하 테스트) 작성을 진행하기 위한 완벽한 자동화 쉘 스크립트들이 준비되어 있습니다. 
내부 로직이나 코드 한 줄을 수정할 필요 없이, 명령어 단 세 개만으로 지표(TPS, P95, P99 등)들을 쉽게 뽑아낼 수 있습니다.

## 사전 요구사항
* 부하 측정 스크립트는 `k6` 엔진을 사용합니다. 사전에 설치해 주세요.
* macOS 설치: `brew install k6`

---

## 단계 1. 테스트 기초 환경 셋업 (테스트 시작 전 최초 1회만 필수)
API 테스트를 위한 코인 발행(Issue)과 마스터 관리자, 송금용 기초 계정을 세팅합니다.

```bash
./run_setup.sh
```
* **동작**: `setup.sh`가 발행→사용자→전환→비밀계좌→전송을 순차 실행하며, 생성된 마스터 정보가 `.state` 파일에 자동 보관 및 활용됩니다.

---

## 단계 2. 단건 응답 속도 확인 테스트 (VU 1명)
API를 **딱 한 번만** 실행하여 단일 사용자의 순수 처리 시간(http_req_duration) 평균과 최대/최소 시간을 확인합니다. 
성능 측정 문서의 **[8. 단건 테스트]** 표를 작성하실 때 사용합니다.

```bash
# 사용법: ./run_single.sh [API명]

# 전체 API 테스트
./run_single.sh

# 개별 API 테스트
./run_single.sh account
./run_single.sh approve
./run_single.sh deposit
./run_single.sh send
./run_single.sh receive
./run_single.sh withdraw
./run_single.sh e2e
```
* **표에 채울 결과 확인법**: 결과 화면의 `http_req_duration` 줄에서 **`avg=`**(평균), **`min=`**(최소), **`max=`**(최대) 수치를 적으세요.
* **특징**: `e2e` 실행 시 1번 셋업 단계에서 기초 자금(1만 단위)이 이체되었던 `ZK_ACCOUNT_ID_01` 단일 공용 계좌를 재사용하여 잔액 부족 없이 1회전 파이프라인을 통과시킵니다.

---

## 단계 3. 다중 / 부하 테스트 (다중 접속, TPS 측정)
지정한 수의 여러 가상 사용자(VU)가 동시 다발적으로 30초 동안 쉴 틈 없이 API를 병렬 호출하여 서버의 한계와 TPS(초당 처리량)를 진단합니다.
성능 측정 문서의 **[9. 다중 테스트]** 및 **[10. E2E 테스트]** 표를 작성하실 때 사용합니다.

```bash
# 사용법: ./run_multi.sh [API명] [VU수]
# VU(동시접속 가상유저 수)에는 10, 20, 30 등을 기입합니다.

# 예시 1: deposit API를 10명의 유저가 동시에 부하시키는 시나리오
./run_multi.sh deposit 10

# 예시 2: E2E 전체 흐름을 30명의 유저가 동시에 부하시키는 파이프라인
./run_multi.sh e2e 30
```

* **표에 채울 결과 확인법**:
  1. **TPS**: `http_reqs` 줄 맨 우측 영단어 옆 숫자
  2. **Error Rate (에러율)**: `checks_failed` 줄에 기록된 퍼센테이지(`%`) 수치
  3. **P95 응답 지연**: `http_req_duration` 줄의 **`p(95)=`** 수치
  4. **P99 응답 지연**: `http_req_duration` 줄의 **`p(99)=`** 수치
